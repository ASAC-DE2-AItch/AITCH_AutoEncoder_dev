# AITCH 전체 그림 — 팀 브리핑 v1.1 (2026-07-06 개정 — v4.5·일정/RnR v3 반영)

> 목적: 회의 전, 지금까지 확정된 **서비스 흐름·설계 근거·각자의 위치**를 한 문서로 공유.
> 상세는 각 문서 참조 (맨 아래 문서 지도). 이 문서는 "왜 이렇게 됐는지"의 요약본.

## 1. 우리가 만드는 것 (한 문단)

Etch 공정 FDC Trace를 실시간 수신해 **C65 불량을 예측**하고(A), **SPC/TTTM으로 이상을 감지**하며(B), 이상 발생 시 AI Agent가 **"진짜 장비 이상인가 / 공정 조건 이탈인가(Recipe) / 기준선이 낡은 건가 / 모두 아닌가"를 판별**해(C) 근거 리포트와 함께 **엔지니어 승인**을 받아 조치까지 연결하는 Human-in-the-Loop 제조 AI 플랫폼. **Recipe 변경은 승인 게이트를 거쳐야만 적용 가능 — 무승인 자동 적용 금지** (v4.5 — 멘토 정정으로 Recipe R2R 부활, 헌법 1-1).

핵심 차별점: 단순 이상 "감지"가 아니라 **판별 → 근거 → 승인 → 조치 → 검증까지 닫힌 루프**를 현업 절차 그대로 구현. 현업은 R2R을 자동(레거시 DB API)으로 돌리지만, 우리는 **설명 가능한 근거 + 승인 게이트**로 감사 추적을 확보한다 (멘토 확인 차별점).

## 3. 설계의 뼈대 — 멘토 확인으로 확정된 것들

| # | 질문했던 것 | 멘토 답 → 우리 반영 |
|---|---|---|
| 1 | 이상 대응 방식 | ~~Recipe 금지(v4.3)~~ → **멘토 정정(v4.5): Recipe R2R 포함** — 조치 4종: Recipe R2R / 실력치 재설정 / 정비 / Disposition (전부 승인 게이트, 무승인 자동만 금지) |
| 2 | TTTM 참조 기준 | golden chamber 지정 안 함 — **동일 공정 챔버 전체 분포(Box Plot) 비교** + 엔지니어 도메인 판단 → fleet median 정량화 + 자기 Qual 스냅샷 보조 |
| 3 | 관리 단위 | **Chamber × Recipe × Step** 확정. Window(과도/정착)는 관리 축이 아니라 지표 추출 계층 |
| 4 | 실력치 갱신 산식 | **이전 3개월 rolling mean ± 3σ, 자동 리캘리브레이션** — EWMA 폐기. λ 차등 이슈 소멸 |
| 5 | 현업 이상 대응 절차 | RTD 배차 제외 → BM 정비 → Sample wafer → 이전 실력치로 계측 — **우리 inhibit → 정비 Agent → Qual 5장 → 4지표 판정과 1:1 대응** |

**실력치 갱신은 3트랙** (4번 반영의 결과):

| 트랙 | 트리거 | 승인 |
|---|---|---|
| 정기 리캘리브레이션 | 주기적, 무이상 | **자동** (폭 상한 ±5% 이내 + 이력 기록 조건 — 헌법 1-1 예외) |
| 이상 대응 재설정 | Incident | **승인 필수** |
| Qual 트리거 재산정 | PM/BM 후 Qual 통과 | **승인 필수** |

세 트랙 모두 같은 산식(rolling mean±3σ), 같은 테이블(`limit_corrections`, trigger_type으로 구분).

범위 밖으로 확정: 골든패스(장비 조합 최적화), dedication, n대화(양산 전개) — 스케줄링/램프업 도메인. 우리 멀티챔버는 "n대화 이후" 상태를 가정.

## 5. 절대 규칙 5개 (PR 차단 사유 — 헌법 요약)

1. **Recipe·실력치·정비·Disposition은 승인 필수** (예외 2개뿐: 정기 리캘리브레이션·Model R2R — 상한 내+기록 조건 자동)
2. **Recipe 무승인 자동 적용 금지** — 튜닝안 생성은 가능, 적용은 승인 게이트 통과 후에만 (v4.5)
3. **데이터 누수 금지** — C65를 feature로 사용 금지, 분할은 Wafer ID 그룹 기준
4. **fdc.alert 단일 채널** + 승인은 Incident당 **동시 pending 1건** (알람당 승인 생성 금지 / 순차 승인은 여러 번 가능 — 헌법 1-4 P3)
5. **센서는 C코드 그대로** (`C11`) — 발표명(DC_Bias)은 화면에서만. `sim_events` 조회 금지(답안지)
