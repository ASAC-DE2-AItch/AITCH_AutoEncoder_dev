## 2. FAIL 진단 (원본 CSV 직접 조사)

### 2-1. FAIL #5 — (C64,C7,C46) 중복 1,456건 = **진짜 병합 손상**

| 관찰 | 값 |
|------|-----|
| 중복 키 | 1,456건 (관련 행 2,912) |
| 완전 중복(65컬럼 동일) | 1,344행 |
| 영향 wafer | 141장 (전체의 0.9%) — **전부 C6_0(정상 생산분)** |
| step 반복 방문 가설 | **기각** (유니크 C39 > 유니크 C7 wafer = 0장) |
| 행/wafer 재확인 | 원본 15행+ wafer(18·20·22·24행) **141장 = 중복 wafer와 정확히 일치** → 데이터 사전의 "24행"은 **복제 부풀림**, 정상 상한은 **14행** |
| 중복 날짜 분포 | 12/04(252)·12/05(1024)·12/14(262)·12/17(254)·12/22(510)·12/23(42)·12/24(40)·01/20(528) — 특정 병합 조각 겹침 구간 |
| 정제 결과 | `(C64,C7,C46)` dedup → **1,456행 제거, 유일성 0, 행/wafer 9~14 정상 복구** |

**판정**: 3분할 병합 과정에서 141개 wafer의 행이 복제 삽입됨. C64 nunique(15,919)는 정상 → **행 복제이지 wafer 중복이 아님**. 대응: dedup을 표준 전처리에 추가(v2).

### 2-2. FAIL #11 — C41 = C10−C39, 95% = **무해한 부동소수점 반올림**

| 관찰 | 값 |
|------|-----|
| 불일치 | 8,191행 (4.97%) |
| 불일치 크기 | 전부 ±0.1~0.9초 (최빈 ±0.1) |
| round 비교 | `round(C41)==C10−C39` → **99.21%** |
| 원인 | C41은 실수 경과시간(예 6.1초), C10−C39는 3초 격자 정수차 — 반올림 불일치 |

**판정**: 병합 손상 아님. 데이터 사전의 "100%"는 train 단독 **정수** 비교였고, 병합본 float에서 오차가 드러난 것. C10(원본 시각)·C41(파생) 모두 **AE 입력 금지**라 실질 영향 0. 대응: 게이트 판정을 반올림 허용(round)으로 수정(v2).

## 3. 추가 발견 — C65 wafer 상수 위반 378장 (v1 게이트엔 없던 항목)

v2 게이트에 C65 상수 검사를 넣는 과정에서 발견:

| 관찰 | 값 |
|------|-----|
| C65 wafer 상수 위반 | **378장 (2.4%)** — dedup과 무관(중복 wafer와 겹침 11장뿐, 원본부터 존재) |
| 위반 wafer의 C65 유니크 수 | 대부분 9~11개 (행마다 다름) |
| C65 `.5` 소수값 | 47,438행 (28.8%) |
| 예시 (C64_0) | step·C42·C41 전부 정상인 멀쩡한 wafer인데 C65만 행별 상이(681·1536·1464.5·…) |

**데이터 사전 모순**: 사전은 "C65는 wafer 내 상수(503~1678)"라 명시. 병합본 2.4%에서 성립하지 않음.

**AE 영향 판정 = 없음**. C65는 불가침 1로 **AE 입력 절대 금지** → 값이 무엇이든 AE 설계·게이트·seg 경계(C33 기준)에 영향 0. 단 **A 트랙 LightGBM(C65 예측)·평가 C65 참조축(F5·F6)·Qual C65 proxy 소관**이므로, **A(김민지)·PM 통보 항목**으로 별도 회부(§5). 원인(병합 오염 vs 구조)은 A 트랙 조사 필요 — 본 REPORT는 검출·기록까지.
