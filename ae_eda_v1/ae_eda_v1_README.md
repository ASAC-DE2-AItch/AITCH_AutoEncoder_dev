# ae_eda_v1 — README (살아있는 문서)

> AE 트랙 P0(환경)+P1(AE 특화 EDA) 노트북. 사용법 · 누적 실험 로그 · 스냅샷 인덱스를 한곳에 모은다.
> 이 파일은 **갱신형**(이름 고정). 고정 스냅샷은 `REPORT/` 참조.

## 노트북 목적
`merged_data.csv`를 ① **믿어도 되는가(무결성 게이트)**, ② AE 입력 **설계 파라미터 5종 확정**. 데이터 사전 결론은 재검증 않고 인용, "병합본에서 성립하는가"만 스팟 체크.

## 사용법
1. 의존성: 프로젝트 루트 `requirements.txt` 설치 (`pip install -r requirements.txt` → `pip freeze > requirements.lock.txt`). EDA 실필수 = numpy·pandas·**pyarrow**. torch는 EDA 무관(모델링부터).
2. `AE/ae_eda_v1/` 에 노트북 두고 로컬 venv 커널로 위→아래 순차 실행. 최초 1회 `merged_data.csv → .parquet` 캐시 생성.
3. 재현: seed=42 고정. 런타임/세트 경계 해시를 REPORT에 기록.

## 설계 원칙 (노트북이 코드로 강제)
- 불가침 1(C65 입력금지·C64 그룹분할) · 2(seg2·C6_1 평가전용, §5-2 관찰만) · 3(스케일러 fit=seg1 train) · 5(과도/정착 분리) · 7(물리신호만·부록A) · 8(출력비움·사용자실행) · 10(모순시 중단·보고)
- **누수 격리**: seg2/C6_1은 §5-2에서 read-only 관찰, 설계 파라미터(§5-3~5-5)는 학습셋(seg1·C6_0)만.
