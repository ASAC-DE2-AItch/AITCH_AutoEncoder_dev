# AE 아키텍처 판단 v1 — 시퀀스 AE(Conv1D/LSTM) vs flatten-MLP (G2 사전 검토)

> **결정 기록 (스냅샷)** · 작성: 세그 / 2026-07-16 · 성격: **P5(G2) 사전 등록** — G2 실측 판정을 대체하지 않음
> 근거: `ae_eda_v3 REPORT_01`(게이트 완전 PASS, DoD4) · `AE_개발계획_v2` §5-4·§7-3·§9 · 데이터_사전 v3.1 · 기획서 v4.5 Won't 목록 · 회의_브리핑 20260714(§1-E C54/C56 재정정)
> 한 줄 결론: **mock을 아무리 증식해도 시퀀스 AE의 전제(시간축 정보량)는 복구되지 않는다(정착 T=2).** v2는 계획 §5-4 분기 그대로 **flatten-MLP로 축소 실행**하고, 주력은 v1 집계 MLP-AE + 과도 모양 지표 보강. 시퀀스 AE는 기각이 아니라 **재검토 조건부 백로그**로 남긴다.

## 0. 질문 2건과 판정 (요약)

| # | 질문 | 판정 | 핵심 근거 |
|---|------|------|----------|
| Q1 | mock 데이터를 더 생성해서라도 1D-CNN + LSTM을 쓰는 게 좋은가? | **반대** | 병목은 표본 수(N)가 아니라 **웨이퍼당 시간축 정보량(T)**. mock 증식은 N만 늘리고 T는 불변(정착 p50=2). 학습용 mock은 "시뮬레이터의 정상"을 학습 — 현업 전이 시 이월 0 |
| Q2 | 현재 데이터 상태에서 flatten-MLP를 쓰는 게 좋은가? | **조건부 채택** (계획 분기 이미 발동) | §5-4 분기 조건 "정착 p50 < 8행 → flatten-MLP" 가 EDA_v3 실측 **p50=2**로 발동. 단 T=2라 v1 집계 피처와 준동치 — **G2에서 공정 판정**하고 미달 시 v1 확정 |

## 1. 판단의 사실 기반 — EDA_v3 실측이 결정적

| 실측 (ae_eda_v3 REPORT_01) | 아키텍처 함의 |
|---|---|
| **Step 4 정착 길이 p50=2 · p90=2 · max=3** (DoD4, T=2) | **Conv1D kernel 3 > 시퀀스 길이 2** — 컨볼루션 연산이 성립하지 않음. 패딩으로 우회해도 수용영역=시퀀스 전체가 되어 "지역 패턴 추출"이라는 존재 이유 소멸 |
| 과도 구간 3행 (점화 첫 9초) | 과도 브랜치를 결합해도 T=3 — LSTM이 학습할 "순서 의존 동역학"이 2~3 스텝. 게이트 메커니즘이 배울 장기 기억 자체가 없음 |
| wafer 전체 9~14행, 3초 샘플링 | 원시 trace가 이미 **요약 수준의 저주파 데이터** — "파형"이 살아있지 않음 |
| 레짐 서명 실재: C17 정착 **−3.30σ**, C12 **+4.14σ** (§3) | AE가 잡아야 할 신호는 실재하나, 그 형태가 **레벨 이동(운전점 변화)**이지 파형 모양 변화가 아님 → **집계 통계(mean/std/min/max)로 포착되는 종류의 신호** |
| 학습셋 seg1·C6_0 = **3,922 wafer** (TRAIN) | 표본 수는 이 규모의 AE에 이미 충분 — "데이터가 부족해서 딥러닝이 안 된다"는 상황이 아님 |

> 요약: 시퀀스 모델을 정당화하는 유일한 조건은 "시간축에 정보가 있다"인데, 실측이 그 전제를 부정한다. 신호는 있으나(레짐 서명) **시간축이 아니라 레벨축에 있다.**

## 2. Q1 — mock 증식 + Conv1D/LSTM: 반대

**① 병목 오진단.** mock 데이터가 늘려주는 것은 웨이퍼 수(N)다. 그러나 시뮬레이터는 실측 세그먼트 위상 리샘플 + 3초 캐던스 + 동일 step 구조로 발행하므로, mock을 10만 장 만들어도 **정착 구간은 여전히 2~3행**이다. Conv/LSTM이 작동할 공간(T)은 1도 늘지 않는다. N은 §1대로 이미 충분하다.

**② 학습용 mock의 구조적 함정.** AE의 "정상"을 mock으로 가르치면 모델이 배우는 것은 **시뮬레이터의 정상 매니폴드**다. 현업(실팹) 도입 시 실데이터로 전면 재학습해야 하고 mock 학습분은 이월되지 않는다. 이 시스템에서 mock(시뮬레이터)의 정위치는 **정답을 아는 평가**다 — 계획 부록 B의 INJECT-TUNE/EVAL(시드 분리 2벌), W7 Scorecard 채점이 정확히 그 역할. **"학습은 실데이터, 채점은 시뮬레이터"** 가 이 플랫폼의 설계 사상이다.

**③ 거버넌스 정합.** 기획서 "하지 않는 것" 목록이 1D-CNN을 기각한 사유가 정확히 **"Trace 형태가 살아있을 때만 의미"** 다. EDA_v3 실측(T=2)은 이 사유를 뒤집는 게 아니라 **강화**한다. 이를 되살리려면 팀 합의 재오픈이 필요한데, 제시할 근거가 반대 방향을 가리킨다.

**④ 현업 도입 3논거.**
- **설명가능성(HITL)**: 이 AE의 점수는 Context Score → Incident → 엔지니어 승인으로 흐른다. 집계 피처 공간에서는 재구성오차를 피처 단위로 분해해 "std_C32_s4 오차 지배적 → 매칭 불안정 의심"처럼 SPC 언어로 번역된다. 시퀀스 잠재공간의 이상은 이 번역 비용이 크다.
- **CT② 재학습 강건성**: 요란 PM마다 재학습하는 운영 루프에서, 작은 통계-MLP는 Phase 2 표본으로 안정적으로 재수렴하고 검증이 쉽다.
- **시스템 정합**: 예측(LightGBM 피처셋)·B의 Nelson/TTTM·Qual σ-갭이 전부 Step×Window 통계 공간에 서 있다. AE만 다른 표현 공간이면 모듈 간 판정 불일치의 원인 추적이 어렵다.

## 3. Q2 — flatten-MLP: 계획 분기 발동, 조건부 채택

**① 분기는 이미 발동됐다.** 계획 §5-4 "정착 p50 < 8행이면 Conv1D 대신 고정 그리드 flatten-MLP" — 실측 p50=2로 조건 충족. EDA_v3 DoD4도 "flatten-MLP(Conv 부적합), T=2"로 확정 기록했다. 즉 Q2는 새 결정이 아니라 **계획의 자기 실행**이다.

**② 단, 기대치는 정직하게.** T=2의 flatten은 채널당 [순번1, 순번2] 두 값 — 집계(mean/std)와 정보량이 준동치다(EDA_v3 §5 한계 2 선기록). v2의 잔여 부가가치는 "순번 위치 보존"(1→2 방향성) 정도로, **G2 기준(v1 대비 L4 저강도 Recall +5%p 또는 L2 평균 +0.03)을 넘기 어려울 가능성이 높다.** 이는 v2를 건너뛸 이유가 아니라 **축소 실행**할 이유다: flatten-MLP 1종 + 과도 브랜치(§6-4), HPO 최소, 동일 분할·동일 하네스·동일 주입 시드(§9-2)로 공정 비교만 수행한다.

**③ 미달 시 처리는 계획 문구 그대로.** "미달 시 v1 확정하고 즉시 P6 진행 (매몰비용 금지)" — 그리고 "**기각도 성과다**: 짧은 trace에선 집계 피처로 충분, 그 자체가 REPORT 결론"(§9-3).

**④ 성능 투자의 우선순위는 v1 보강이 더 효율적.** 시퀀스 모델이 잡아줄 것으로 기대했던 "모양 정보"의 정수는 이미 EDA가 스칼라 지표로 추출해뒀다:
- `C11_wf_min` — 점화 스파이크 최심점 (레짐 프록시. 단, v3 실측 +0.21σ로 약함 — P4에서 재확인)
- C18 과도 피크 (매칭 과도 프로파일 0.01→4.16→0.04의 요약)
- C62 점화 반전 델타 (첫 샘플 −3553 → 정착 +1800)
- **C54/C56 과도 wafer-mean** — 7/13 재정정(정착→과도 복귀, B 자가 발견·PM 승인) 반영
- 정착 도달 속도·완결성 피처 (§5-4의 길이 피처)

3~5개 점으로 구성된 "모양"은 이런 지표 몇 개로 거의 손실 없이 요약된다 — 이것이 실팹 FDC의 표준 관행(trace → summary indicator → 감시)이기도 하다.

## 4. 현업 도입 방향성 (결론)

**주력 = v1 집계 MLP-AE (+과도 모양 지표 보강), v2 = flatten-MLP 축소 실행 후 G2 공정 판정.** 이 구성은 실팹 FDC 관행과 표현 공간이 일치해 도입 장벽이 최소이고, HITL 설명·CT② 재학습·타 모듈 정합의 3요건을 모두 만족한다.

**시퀀스 AE(Conv1D/LSTM)는 기각이 아니라 재검토 조건부 백로그** — 아래 트리거 중 하나가 실증되면 재오픈한다 (§9-3 백로그 조건과 정합):

| 트리거 | 의미 |
|---|---|
| T1. 샘플링 고주파화 — ≥1Hz, step당 50행 이상 | "trace 형태가 살아있다"는 기획서 조건이 실제로 성립 |
| T2. Scorecard에서 **모양-전용 미탐** 실증 | 집계+모양지표로 못 잡는데 파형에는 있는 이상이 존재함이 증명됨 |
| T3. 시뮬레이터가 더 긴 trace를 발행하도록 스펙 변경 | 데모 세계 안에서 T가 복구됨 |

mock 데이터의 역할은 그때도 **학습이 아니라 평가**다(INJECT 시드 분리·Scorecard).

## 5. 정직한 한계

1. **이 문서는 G2를 대체하지 않는다** — 사전 등록(pre-registration)이다. P5 실측 비교(동일 하네스)는 계획대로 수행하고, 이 문서의 예측(G2 미달 가능성 높음)이 틀리면 그 결과가 우선한다.
2. C11_wf_min 레짐 반응이 v3에서 약했다(+0.21σ vs F14 +6.56σ) — 과도 지표 보강의 실효는 P4 학습에서 확인 필요.
3. seg2(레짐 홀드아웃) 미접촉 상태의 판단 — "정상(seg1) 지형" 기준이다.
4. 현업 논거(설명가능성·재학습 강건성)는 정성 판단 — NFR 실측(G3)에서 수치로 보강한다.

## 6. 반영 사항

- `AE_개발계획_v2` §9는 **수정 불요** — 분기 조건이 이미 이 결론을 내장. P5 착수 시 "분기 발동(p50=2)" 명시만 추가.
- P2(피처 명세)에서 §3-④의 과도 모양 지표를 v1 피처 후보로 등재.
- G2 판정 REPORT 작성 시 이 문서를 선행 문서로 인용 (예측 적중/불일치 대조).
